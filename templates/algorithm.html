Algorithm


The driving idea behind the algorithm that generates this vocabulary is the one of Context-Free Grammar. The majority of how this algorithm and the idea behind this language occurred to me during a lecture of my Computer Architecture class where we were being introduced to the basics of compilers. The general idea for a compiler for a programming language is that you have a tree-like structure which you can traverse bottom-to-top, in order to validate whether or not the syntax is correct and to generate more low-level code that can be interpreted by a computer (eg. C to Assembly).

But, if we traverse the tree top-down, we can do the inverse of validation -- generation. That's the guiding methodology behind the construction of vocabulary in krn. The algorithm starts with the pre-defined three-letter root words in the Dictionary, and traverses a tree downwards to create all the possible words that root can produce.
